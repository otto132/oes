# Lead & Pipeline Redesign v2 — Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Redesign the lead → account → opportunity flow so qualified leads always become pipeline deals, with clean 7-stage opportunity lifecycle, bidirectional traceability, and FIUAC→health seeding.

**Architecture:** Prisma schema changes (new enums, FK, fields) → types/constants → Zod validation → adapters → API routes → React Query hooks → API client → seed data → tests → UI pages. Each layer builds on the previous one. TDD approach: write failing tests first, then implement.

**Tech Stack:** Next.js 14, Prisma (PostgreSQL), React Query, Zod, Vitest, TypeScript

**Spec:** `docs/superpowers/specs/2026-03-15-lead-pipeline-redesign-v2-design.md`

export async function GET(req: NextRequest) {
  const session = await auth();
  if (!session?.user?.id) return unauthorized();
  const scoped = scopedDb(session.user.id, (session.user as any).role ?? 'VIEWER');

  const pagination = parsePagination(req);
  const showPaused = req.nextUrl.searchParams.get('paused') === 'true';

  const where = showPaused
    ? { stage: 'Paused' as const }
    : { stage: { notIn: ['Converted', 'Disqualified', 'Paused'] as const } };

  const leads = await scoped.lead.findMany({
    where,
    include: { owner: true },
    orderBy: { createdAt: 'desc' },
    take: pagination.limit + 1,
    ...(pagination.cursor ? { cursor: { id: pagination.cursor }, skip: 1 } : {}),
  });

  const { data, meta } = paginate(leads, pagination.limit);
  return NextResponse.json({ data: data.map(adaptLead), meta });
}
